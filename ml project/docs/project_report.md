# Retail Demand Forecasting Project Report

## 1. Project Overview
This project builds an end-to-end demand forecasting solution for retail stores using the Rossmann Store Sales dataset. It includes:
- data loading and preprocessing
- exploratory data analysis
- multiple models (Linear Regression and Random Forest)
- a Streamlit dashboard for interactive forecasting
- GenAI-driven narrative insights and anomaly explanations

## 2. Business Objective
The goal is to forecast daily sales so store managers can optimize inventory, staffing, and promotions. Predictions help reduce stockouts, improve staffing efficiency, and make promotional planning more reliable.

## 3. Dataset Summary
The dataset includes:
- `train.csv`: historical sales and store activity
- `store.csv`: store metadata such as store type, assortment, and competition

Key features used in modeling:
- Store ID
- Day of week
- Promotion status
- State holiday
- Store type and assortment
- Competition distance
- Year, month, day, and week of year

## 4. Data Preprocessing
The preprocessing pipeline performs:
- missing value handling for competition and promotion fields
- date feature extraction (year, month, day, week)
- filtering to open store days with positive sales
- categorical encoding for `StoreType`, `Assortment`, and `StateHoliday`

This transforms raw retail sales data into a clean, structured dataset fit for regression.

## 5. Exploratory Data Analysis
The EDA explores:
- sales distribution
- sales impact from promotion status
- seasonal monthly trends
- correlation among features

Visual outputs are generated into `notebooks/` as PNG files for reporting and presentation.

## 6. Modeling Approach
Two models are trained:
- Linear Regression: baseline explainable model
- Random Forest Regressor: stronger non-linear forecast model

Training includes a randomized 80/20 train/test split and evaluation using:
- RMSE (Root Mean Square Error)
- MAE (Mean Absolute Error)

## 7. Results and Insights
The project provides:
- forecast values from both models
- demand comparison metrics
- AI-powered narrative interpretation for business users

The dashboard enables scenario planning by adjusting store ID, promotion, day of week, and holiday conditions.

## 8. Deployment Readiness
Deployment artifacts created:
- `Dockerfile` for containerized deployment
- `Procfile` for cloud platform compatibility
- `requirements.txt` with runtime dependencies
- `README.md` and `docs/deployment_guide.md` for setup and hosting

## 9. Limitations
Current limitations include:
- simulated AI explanations depend on Anthropic API access
- the dashboard uses simplified default store profile values for interactive simulation
- real-time prediction currently depends on pre-trained models generated by the pipeline

## 10. Next Steps
Future improvements:
- add a production-grade forecasting API service
- use time-series-specific models (Prophet, XGBoost, LSTM)
- add feature importance and explainability graphs
- integrate real store metadata and demand signals from external sources

## 11. Conclusion
This project delivers a complete retail demand forecasting solution with a deployable dashboard, model training workflow, and AI-powered business insight layer. The documentation and deployment assets make it ready for local use and cloud hosting.
